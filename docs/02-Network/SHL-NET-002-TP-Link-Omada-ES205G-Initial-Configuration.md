# 02 - Objective

Configure the TP-Link Omada ES205G managed switch for the Sanji Homelab infrastructure.

The switch will act as the central Layer 2 device connecting all infrastructure equipment.

---

# 03 - Context

The TP-Link Omada ES205G is deployed as the core switch of the Sanji Homelab.

It is connected to RTR-01 (TP-Link WR802N operating in WISP mode) and distributes network connectivity to all servers and network devices.

---

# 03.5 - Prerequisites

- TP-Link Omada ES205G
- Ethernet cable
- Administration workstation
- Access to the switch Web Interface
- Administrator credentials
- RTR-01 already configured

# 08 - Troubleshooting

| Symptom | Possible Cause | Solution |
|----------|----------------|----------|
| Cannot access Web Interface | Incorrect IP configuration | Verify management IP |
| No network connectivity | Ethernet cable disconnected | Verify physical connections |
| Switch unreachable | Wrong subnet | Verify gateway and subnet mask |
| Configuration lost | Configuration not saved | Save running configuration |

---

# 09 - Rollback

If configuration fails:

1. Press and hold the RESET button for approximately 10 seconds.
2. Wait for the switch to reboot.
3. Restore factory settings.
4. Reconfigure the switch.

# Risk Assessment

| Risk | Impact | Mitigation |
